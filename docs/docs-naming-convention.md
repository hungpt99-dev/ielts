# Docs Naming Convention

## Rule

All documentation files in `docs/` **MUST** use **kebab-case** with `.md` extension.

## Pattern

```
lowercase-kebab-name.md
```

## Examples

| ✅ Correct | ❌ Incorrect |
|---|---|
| `ai-architecture.md` | `AI_ARCHITECTURE.md` |
| `contribution-guide.md` | `ContributionGuide.md` |
| `database-schema.md` | `database_schema.md` |
| `product-overview.md` | `productOverview.md` |
| `local-first-design.md` | `local_first_design.md` |
| `troubleshooting.md` | `Troubleshooting.md` |

## Rules

- **Casing**: all lowercase
- **Separator**: hyphens (`-`) between words
- **Extension**: `.md`
- **Single-word files**: no separator needed (e.g., `schema.md`, `deployment.md`)
- **Subdirectories**: may contain their own `README.md` or kebab-case files

## Scope

Applies to all `.md` files directly in `docs/` and its subdirectories.

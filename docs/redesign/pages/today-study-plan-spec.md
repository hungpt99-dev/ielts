# IELTS Journey — Today's Study Plan Page Specification

## Page Purpose

The Today's Study Plan page is the user's daily learning workspace. It answers "What exactly do I need to study today?" by presenting a detailed, actionable checklist of tasks generated by the AI Study Plan. Unlike the dashboard (which provides an overview + mission), this page is the execution view — where the user works through their plan, marks tasks complete, adjusts timing, and receives AI Tutor guidance per-task.

**Core promise:** "Open this page, follow the checklist, and you've completed today's IELTS study."

## User Goal

- See every task they need to do today at a glance
- Understand today's goal, estimated time, and priority
- Check off tasks as they complete them
- Get AI Tutor explanation or help on any task
- Adjust or skip tasks if needed
- Feel a sense of accomplishment when all tasks are done
- Know where today fits in the larger study roadmap

## Current UX/UI Problems

Based on analysis of the current implementation (`apps/web/src/features/study-plan/StudyPlan.tsx`, `apps/web/src/pages/DailyPlan.tsx`, `apps/web/src/features/studyPlan/components/DailyPlanList.tsx`):

1. **No dedicated "Today" view** — The current `/plan` page defaults to a "Phases" tab showing all phases collapsed. The "This Week" tab shows the week but requires scrolling. There is no single-purpose "Here's today's plan" page.

2. **Today is buried in the plan structure** — To see today's plan, the user navigates: Phases tab → expand phase → scroll through weeks → find today. Too many steps.

3. **DailyPlan.tsx is a generic task manager** — The existing `DailyPlan.tsx` page is a general task CRUD interface with filters, sorting, and manual task creation. It doesn't surface the AI-generated plan's structure (main goal, per-skill tasks, AI tutor note, completion checklist).

4. **No skill task categorization** — The new system's `DailyPlanItem` model has distinct fields: `listeningTask`, `readingTask`, `writingTask`, `speakingTask`, `vocabularyTask`, `grammarTask`, `reviewTask`. The current StudyPlan page flattens all tasks into a single task list with category badges, losing the intentional structure.

5. **No "AI Tutor Note" prominence** — `DailyPlanItem.aiTutorNote` exists in the model but is hidden behind a "Show AI note" toggle in `DayCard`. It should be surfaced as part of the day's guidance.

6. **No completion checklist visibility** — `DailyPlanItem.completionChecklist` exists but is toggled behind a "Show checklist" button. Today's plan should show the checklist inline.

7. **No progress celebration** — When all tasks are complete, there is no celebration, animated transition, or encouraging message.

8. **No "skip/adjust" UX** — Users can change the day status via a dropdown (`not-started`, `in-progress`, `completed`, `skipped`, `partially-completed`) but there's no friction-minimal way to reschedule a task to tomorrow or skip just one task.

9. **No study time timer or tracking** — Users have no way to track how long they actually spend on each task vs the estimate.

10. **No connection to other pages** — Today's plan doesn't link directly to the practice pages for each skill. E.g., clicking a Reading task should ideally open the Reading Practice page with that passage.

11. **Mobile experience is cramped** — The task rows, badges, time estimates, and checkboxes are all in a tight layout on mobile.

12. **Today's goal not emphasized** — `DailyPlanItem.mainGoal` is just text in the card. It should be a prominent, motivating headline.

## Main Sections

### 1. Today's Goal Header

**Purpose:** Present the day's main objective prominently so the user knows what to focus on.

**Content:**
- Back link/button: "← Dashboard" navigates to `/dashboard`
- Large goal headline: `DailyPlanItem.mainGoal` written as an actionable statement
- Metadata row:
  - Date with weekday (e.g., "Tuesday, June 16")
  - Day number of plan (e.g., "Day 45 of 90")
  - Phase name badge (e.g., "Skill Improvement" phase)
- Info chips row:
  - Estimated total time (clock icon + minutes)
  - Priority badge (`--color-warning` for high, `--color-info` for medium, `--color-muted` for low)
  - Difficulty badge (`--color-danger` for hard, `--color-warning` for medium, `--color-success` for easy)
- Overall progress bar: Animated, with count label "4/6 tasks"
- Primary CTA: [Continue Studying] — scrolls to first incomplete task, or shows "Resume" if partially done
- Secondary CTA: [Ask AI Tutor] — opens AI Tutor with today's context

**States:**
- **Not started (not-started):** Show progress bar at 0%, CTA = "Start Studying"
- **In progress (in-progress):** Show partial progress bar, CTA = "Continue Studying"
- **Completed (completed):** Replace with celebration card (see below)
- **Skipped (skipped):** Show muted header with "Skipped" badge, option to undo

### 2. AI Tutor Note Card

**Purpose:** Surface the AI-generated tip or guidance for today's study session.

**Content:**
- Light bulb or sparkle icon
- Label: "Today's Tip" or "AI Tutor Note"
- Text: `DailyPlanItem.aiTutorNote` — rendered as friendly, conversational guidance
- Optional: "Ask AI about this" quick-action link

**States:**
- **Has note:** Show card with content
- **No note:** Card hidden (no empty space)
- **Loading:** Skeleton text block

### 3. Completion Checklist

**Purpose:** Show the pre-defined checklist items so the user knows what "done" means for today.

**Content:**
- Section title: "Today's Checklist"
- List of `DailyPlanItem.completionChecklist` items as checkable rows
- Each item: circular checkbox + text description
- Checked items: strikethrough, muted color
- Progress summary at bottom: "3/4 items checked"

**States:**
- **Has items:** Show checklist
- **No items:** Section hidden
- **All checked:** "All checked!" label with green checkmark

### 4. Skill Task Cards

**Purpose:** List each skill-specific task with status, time, and AI Tutor access.

**Content:**
One card per present skill task, in the order: Listening, Reading, Writing, Speaking, Vocabulary, Grammar, Review. Each card contains:

- **Skill icon + label** (color-coded by skill):
  - 🎧 Listening (cyan/green)
  - 📖 Reading (blue)
  - ✍️ Writing (amber)
  - 🎤 Speaking (teal)
  - 📚 Vocabulary (purple)
  - 🔤 Grammar (pink)
  - 🔄 Review (slate)
- **Task title**: `task.title`
- **Task description**: `task.description` (secondary text, truncated if long)
- **Estimated time badge**: Clock + minutes
- **Checkbox**: Native-styled circular checkbox with smooth transition
- **Checked state**: Subtle green left border, strikethrough title, reduced opacity
- **Action buttons:**
  - 🧠 [Ask AI Tutor] — calls `handleAskAiTutor` with task context, shows explanation inline
  - 📝 [Notes] — expandable inline text area for personal notes
  - 🔗 [Open Practice] — navigates to relevant practice page with task context (when applicable)
- **AI explanation inline**: When AI Tutor responds, show explanation box below with skill-colored left border and tutor icon

**States:**
- **Loading:** Skeleton cards matching task structure
- **Empty (no tasks):** Section shows "No tasks for this skill today" (only appears when no tasks at all)
- **AI explaining:** Show spinner in Ask AI button, disable until response
- **Notes open:** Expand card height, show textarea
- **Task completed:** Card visually shifts to completed state with animation

### 5. Task Action Menu (Per Task)

**Purpose:** Allow the user to adjust, skip, or reschedule individual tasks without leaving the page.

**Content:**
- Three dots (kebab) menu or expandable actions on each task card
- Actions:
  - **Skip this task** — marks task as skipped, moves to bottom with muted styling
  - **Move to tomorrow** — reschedules task to next day (calls updateDailyPlan)
  - **Reduce time** — opens a quick picker to lower estimated minutes
  - **Ask AI to explain** — same as Ask AI button
- Confirmation for destructive actions: "Skip this task?" with [Cancel] [Skip]

### 6. Weekly Context Strip

**Purpose:** Show how today fits into the current week.

**Content:**
- Section title: "This Week" or weekly focus (from `DailyPlanItem` week's focus)
- Mini week progress: 7 small pill indicators (Mon-Sun), each filled/in-progress/empty based on day status
  - Completed day: filled green
  - Today: filled primary with pulse animation
  - Future day: outlined
  - Skipped: outlined with diagonal line
- Stats row: "X/Y days done" · "Z min studied / target"
- [View Full Week] action → scrolls to week view or navigates to `/plan?tab=week`

**States:**
- **No week data:** Section hidden
- **First day of week:** "This is the first day of your study week!"

### 7. Celebration Card (All Complete)

**Purpose:** Provide a satisfying end-of-session experience when all tasks are done.

**Content:**
- Large celebration icon: 🎉 or confetti animation
- Title: "Great job, {name}!"
- Summary: "You completed today's study plan! X/Y tasks done in Z min."
- Stats: Actual time vs estimated, streak update, next target
- Actions:
  - [View Tomorrow's Plan] → navigates to tomorrow's plan (or `/plan?date=tomorrow`)
  - [Chat with AI Tutor] → opens AI Tutor with "I finished today's plan!" context
  - [Review What I Learned] → opens review summary

**Animation:**
- Confetti-like particle animation (CSS-only, lightweight)
- Progress bar animates to 100%
- Cards sequentially fade to completed state

## Empty State

**When:** The user has no study plan generated, or today has no tasks scheduled.

**Visual:**
- Friendly illustration (a calendar with an empty page / a person looking at a blank schedule)
- Title: "Nothing planned for today"
- Description: "You don't have any tasks scheduled for today. Either your plan hasn't been generated yet, or today is a rest day."
- Actions:
  - [Generate My Study Plan] → `/plan` with generate modal open
  - [Explore Something New] → `/practice`
  - [Chat with AI Tutor] → `/ai-tutor`

**Sub-states:**
- **No plan at all:** "You haven't created a study plan yet. Let's build one!"
- **Rest day:** "Today is a rest day in your plan. Enjoy the break!"
- **Study plan finished:** "Congratulations! You've completed your study plan up to your exam date."

## Error State

**Visual:**
- Centered error illustration
- Title: "Couldn't load today's plan"
- Description: Friendly error message (not technical)
- Actions:
  - [Try Again] — reloads data
  - [Go to Dashboard] — `/dashboard`

**Recovery:**
- If plan data loads but a single task fails: Show other tasks, show inline error on failed task
- If AI Tutor explanation fails: Show "AI Tutor unavailable" inline, allow retry per task
- If all data fails: Full error state

## AI Tutor Integration

### Per-Task AI Explanation

Each task card includes an "Ask AI Tutor" action that:
1. Calls the AI with task context (title, description, skill, phase, day number)
2. Shows loading state on the button
3. Displays the AI explanation inline below the task
4. Saves the explanation to `task.notes` so it persists

### Today's AI Tutor Note

The `DailyPlanItem.aiTutorNote` is displayed prominently as a tip card. Examples:
- "Today's Tip: Focus on listening for signposting words..."
- "Today's Tip: Pay extra attention to Writing Task 2 structure..."

### Context-Aware Suggestions

When the user clicks "Ask AI Tutor" from the hero CTA or a task card, the AI Tutor page should open with context pre-populated:
- Which skill the user is working on
- Today's main goal
- Which tasks are done/remaining
- Any recent mistakes related to today's skills

### Integration Points

| Entry Point | Opens | Context |
|---|---|---|
| Hero "Ask AI Tutor" | AI Tutor page/drawer | Full today's plan context |
| Per-task "Ask AI" | Inline explanation | Task-specific context |
| Celebration card "Chat" | AI Tutor page | "Just finished today's plan" |

## Accessibility Notes

- All checkboxes: Use native `<input type="checkbox">` with custom styling, not `<div>` + `aria`
- Task cards: `aria-label` summarizing task (e.g., "Listening task: Passage 1, 20 minutes, not started")
- Progress bar: `role="progressbar"`, `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax={totalTasks}`
- AI Tutor note: `role="status"` with `aria-live="polite"` for dynamic content
- Action menus: Use native `<dialog>` or `role="menu"` with keyboard navigation
- Skip/adjust: Confirm destructive actions with `role="alertdialog"`
- Loading: `aria-hidden="true"` on skeletons, `role="status"` with `aria-label="Loading today's plan"`
- Celebration: `role="status"` with `aria-live="assertive"` when all tasks complete
- Touch targets: Minimum 44x44px on mobile for all interactive elements
- Focus order: Goal header → AI note → Checklist → Tasks in display order
- Color: Do not rely solely on color for task status — use strikethrough + checkmark icon + reduced opacity
- Keyboard: Tab through tasks, Space to toggle checkbox, Enter to expand actions

## Design Notes (Inspired by Golovko's Reference)

1. **Card-based task layout**: Each skill task should be a distinct rounded card with a left accent border in the skill's color (like the reference's subject cards). This makes the page feel like a collection of learning activities, not a flat to-do list.

2. **Progress as a visual spine**: The reference uses a prominent progress indicator at the top of daily views. The goal header's progress bar should be thick (h-3), rounded, and animate smoothly as tasks are checked off.

3. **Skill icon consistency**: Use consistent colored icon circles for each skill (as the reference does for subjects). These same icons appear on the dashboard, study plan, and practice pages — creating a visual language.

4. **Friendly microcopy**: Replace "Task 1: Listening Passage" with "🎧 Complete Listening Section 1" and include encouraging notes: "You've got this!" when 50% done, "Almost there!" at 80%.

5. **Time blocks as visual weight**: Each task card should visually communicate its time commitment. A 25-minute Reading card should feel more substantial than a 5-minute Review card. Use subtle height variation or emphasis.

6. **White space generosity**: The reference uses abundant white space between sections. Each task card should have comfortable internal padding (`p-4` to `p-6`) and consistent gap between cards (`gap-3`).

7. **Subtle shadows**: Cards use `--shadow-sm` by default, elevating to `--shadow-md` on hover/interaction. The reference shows this layered shadow effect beautifully.

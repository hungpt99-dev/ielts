# Vocabulary

## Purpose

Store and manage the learner's personal vocabulary collection with spaced-repetition review scheduling.

## Canonical Model

**VocabularyEntry** (defined in `apps/web/src/models/index.ts`):

| Field | Type | Description |
|---|---|---|
| `id` | string | UUID |
| `word` | string | The vocabulary word |
| `meaning` | string | Definition in English |
| `meaningVi` | string | Vietnamese translation |
| `pronunciation` | string | IPA or phonetic spelling |
| `partOfSpeech` | string | e.g. `'noun'`, `'verb'` |
| `exampleSentence` | string | Usage example |
| `collocations` | string[] | Common word pairings |
| `synonyms` | string[] | Similar words |
| `antonyms` | string[] | Opposite words |
| `wordFamily` | string[] | Related word forms |
| `difficulty` | `'easy' \| 'medium' \| 'hard'` | Learner-assigned difficulty |
| `status` | `'new' \| 'learning' \| 'reviewing' \| 'mastered'` | Learning stage |
| `cefrLevel` | `'' \| 'A1' \| 'A2' \| 'B1' \| 'B2' \| 'C1' \| 'C2'` | CEFR language level |
| `ieltsRelevance` | `'' \| 'low' \| 'medium' \| 'high'` | Relevance to IELTS |
| `tags` | string[] | User-defined tags |

## Spaced Repetition

**VocabReviewEntry** tracks each word's review schedule:

| Field | Type | Description |
|---|---|---|
| `vocabularyId` | string | FK to vocabulary entry |
| `interval` | number | Days until next review |
| `easeFactor` | number | SM-2 ease factor (default 2.5) |
| `repetitions` | number | Consecutive correct reviews |
| `nextReviewDate` | ISOString | Scheduled review date |
| `history` | `{ date, rating }[]` | Review history |

**Algorithm** (`apps/web/src/utils/spaced-repetition.ts`):

SM-2-based with four ratings:

| Rating | Effect |
|---|---|
| `'again'` | Reset: interval=1, EF−0.2 (min 1.3), reps=0 |
| `'hard'` | Interval×1.2, EF−0.15 |
| `'good'` | Standard SM-2 progression: 1→1→6→interval×EF |
| `'easy'` | Interval×EF×1.3, EF+0.15 |

`getDailyReviewQueue()` returns all words whose `nextReviewDate` ≤ today, excluding `'mastered'` words.

## Lifecycle

```
word saved (status='new') → first review → 'learning'
    → spaced repetition reviews → 'reviewing'
    → consistent correct reviews → 'mastered'
```

- Words can be added manually, via the extension's save feature, or imported from public APIs (Wiktionary, Datamuse, Tatoeba).
- Review sessions can be generated by the learning engine (`VocabularySkillModule`).

## Persistence

- `vocabulary` table (IndexedDB, Dexie) via `VocabularyRepository`.
- `vocabularyReviews` table via `VocabReviewRepository`.
- Extension: separate `vocabulary` store in extension's IndexedDB.

## Events

- `'vocabulary_saved'`: new word added.
- `'vocabulary_reviewed'`: review completed (carries `correct` flag).
- `'vocabulary_mastered'`: word reached `'mastered'` status.
- `'vocabulary_review_due'`: batch of words due for review.

## Invariants

- Each word has exactly one `VocabReviewEntry`.
- Status transitions: `new → learning → reviewing → mastered` (can regress from any to `learning` via `'again'` rating).
- `easeFactor` is clamped to [1.3, ∞).

## Migration Considerations

- The `vocabulary` and `vocabularyReviews` tables have been stable since v1.
- CEFR level and IELTS relevance fields were added later; existing entries use empty string defaults.

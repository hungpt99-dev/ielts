# Mistakes

## Purpose

Capture, categorise, and track learner mistakes across all skills. Mistakes drive review sessions, tutor recommendations, and targeted practice.

## Canonical Model

**MistakeEvidence** (`@ielts/learning-engine`):

| Field | Type | Description |
|---|---|---|
| `id` | string | UUID |
| `skill` | `IELTSSection` | Skill where mistake occurred |
| `category` | string | e.g. `'subject-verb agreement'`, `'article usage'` |
| `subcategory` | string \| undefined | Finer-grained classification |
| `originalResponse` | string | What the learner wrote/said |
| `correctedResponse` | string | The correct version |
| `explanation` | string | Why it was wrong |
| `sourceExerciseId` | string | Exercise that revealed the mistake |
| `sourceQuestionId` | string | Specific question |
| `occurredAt` | string | ISO timestamp |
| `recurrenceCount` | number | How many times this pattern occurred |
| `severity` | `'minor' \| 'moderate' \| 'severe' \| 'critical'` | Escalates with recurrence |
| `reviewStatus` | `'unreviewed' \| 'reviewed' \| 'mastered'` | Learner's review progress |
| `confidence` | number | AI confidence in detection |

**Build flow:**

`buildMistakeEvidence()` in `domain/services/mistake-evidence-builder.ts`:
1. Checks for similar existing mistakes (same skill + category + response text).
2. If found, increments `recurrenceCount`.
3. Calculates severity: 0→`minor`, 1→`moderate`, 3→`severe`, 5+→`critical`.
4. Returns new `MistakeEvidence`.

**Recurrence detection:**

`detectRecurrencePattern()` groups mistakes by `skill:category` and returns patterns with count ≥ 2. Used by the AI tutor to identify recurring problems.

## Storage

- **Primary table**: `mistakes` in IndexedDB (Dexie), via `MistakeRepository`.
- Mistake entries are typed as `MistakeEntry` in storage and `MistakeEvidence` in the learning engine.
- **Extension**: separate `mistakes` store in extension's IndexedDB (`ielts-journey-extension`, 5 stores).

## Lifecycle

```
Mistake detected (during evaluation) → stored in mistakes table
    → reviewed by learner (status → 'reviewed')
    → mastered after repeated success (status → 'mastered')
```

- Mistakes are created during `answer_evaluated` events.
- Mistake review sessions are generated by `generateMistakeReview()`.
- Tutor memory tracks mistake patterns (see tutor-memory.md).

## Events

- `'mistake_detected'`: first occurrence.
- `'mistake_repeated'`: recurrence updated.
- Both events carry `mistakeId`, `skill`, `category`, and `recurrenceCount`.

## Invariants

- A mistake is uniquely identified by `skill + category + originalResponse`.
- `recurrenceCount` is monotonically increasing.
- Severity is derived from `recurrenceCount`, never set directly.

## Duplicate Models

- The web app's `MistakeEntry` has fields like `text`, `source`, `status` ('new'/'reviewed'/'resolved'). The learning engine's `MistakeEvidence` is richer. Mappers in `@ielts/shared` (`mapLegacyMistakeToEvidence`) convert between them.
- The `mistakes` table is shared — both legacy and new code write to it.

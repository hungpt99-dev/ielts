# Study Plan

## Purpose

Generate a structured IELTS preparation roadmap — a phased, week-by-week plan with daily tasks — and track task completion. The plan adapts to the user's profile, exam date, and progress.

## Canonical Model

Tasks are stored in the `tasks` table (IndexedDB via Dexie). Roadmap structure (phases, weeks, days) is computed by `planConverter.ts` from the study plan output.

**Task entry (`TaskEntry`):**

| Field | Type | Description |
|---|---|---|
| `id` | string | UUID |
| `title` | string | Task name |
| `description` | string | Details |
| `category` | `TaskCategory` | e.g. `'Vocabulary'`, `'Reading'`, `'Writing Task 1'` |
| `date` | ISOString | Scheduled date |
| `isDone` | boolean | Completion flag |
| `completedAt` | ISOString \| null | When completed |
| `estimatedMinutes` | number | Planned duration |
| `accuracy` | number \| null | Score from associated session |

**Study plan domain (`StudyPlan`):**

- Generated by `DailyPlanEngine` (in `@ielts/learning-engine/daily-plan`) using `AiPlanOrchestrator`.
- Based on `UserProfileInput`: target band, exam date, availability, weak skills, past mistakes.
- Outputs a `StudyPlan` containing phases, weeks, daily tasks with `StudyTask` entries.
- `PlanEngineIntegration` bridges between app settings and the engine's `UserProfileInput`.
- `PlanRegenerator` handles plan updates when profile changes.

## Lifecycle

1. **Generate**: user completes onboarding → `DailyPlanEngine.generate()` → tasks written to IndexedDB.
2. **View**: roadmap UI reads tasks, groups by date, displays phases/weeks/days via `planConverter`.
3. **Complete**: user marks task done or completes a learning session linked to the task → `isDone = true`.
4. **Regenerate**: profile change triggers `PlanRegenerator` → tasks are adjusted.

## Invariants

- Each task has exactly one `date`.
- Tasks are ordered by date within the plan.
- A completed task may reference a `LearningSession` via the roadmap.
- Tasks are not deleted on regeneration; old tasks are superseded.

## Ownership

Owned by `@ielts/learning-engine` (plan generation logic). Persistence in `@ielts/storage` (tasks table). UI rendering in the web app's roadmap feature.

## Persistence

- Tasks: IndexedDB `tasks` table, `TaskRepository`.
- Plan metadata: `customStudyPlans` table (active plan flag).
- No server-side storage; all data is local.

## Events

- `'task_completed'`, `'task_missed'`, `'task_rescheduled'` via `SharedLearningEvent`.
- `'roadmap_generated'`, `'roadmap_updated'` after plan changes.
- Tutor proactive messages like `'daily_plan_ready'`, `'roadmap_task_available'`.

## Duplicate Models

- The web app has legacy `AppSettings` with `dailyStudyMinutes`, `preferredSchedule`, `studyGoal`.
- The learning engine's `UserProfileInput` is the canonical profile for plan generation; `buildNormalizedProfile` maps from legacy settings.

## Migration Considerations

- Plan regeneration must preserve completed tasks.
- The `tasks` table schema has remained stable since v1 (only index additions).

# Automation Design: Video Helper Feature

## Current State Summary

The video helper flow requires **7-10 manual steps**: navigate to YouTube → badge appears → click badge → popup opens → copy transcript from YouTube description/CC → paste into textarea → click each AI tab (Vocabulary, Summary, Questions, Shadowing) one by one → wait for each to complete → review results → click "Save Video & Vocabulary". The transcript must be sourced manually — there is no YouTube Data API or transcript API integration. AI analysis is sequential per tab with no batch mode. Video metadata is limited to title, URL, and ID extracted from the DOM. No background saving, no queue, no auto-detection of new videos on channels the user follows.

## Proposed Automation Enhancements

### 1. Automatic YouTube Transcript Fetching

| Change | Location | Description |
|--------|----------|-------------|
| YouTube transcript API client | `packages/ai/src/services/youtube.ts` (new) | New package module that fetches captions via `youtube-transcript-api` (or inline fetch against `youtubetranscript.com` API). Supports auto-generated and manual captions. Falls back gracefully if no captions available. |
| Transcript fetch in content script | `content-script/videoHelper.ts` | When a YouTube video page is detected, auto-fetch the transcript in the background using the content script's fetch privilege. Cache result in session storage to avoid re-fetch on popup open. |
| Transcript auto-populate | `popup/components/VideoHelper.tsx` | When popup opens and transcript was already fetched by content script, auto-populate the transcript field and hide the manual paste section. Show a badge: "Transcript fetched automatically (XX captions)". |
| Manual transcript fallback | `popup/components/VideoHelper.tsx` | If auto-fetch fails (no captions), keep the existing manual paste UI with a clear message: "No captions found. Paste transcript manually or try a different video." |

**Implementation Pattern** — new file `packages/ai/src/services/youtube.ts`:

```typescript
interface TranscriptFetcherConfig {
  proxyUrl?: string
}

interface TranscriptSegment {
  text: string
  duration: number
  offset: number
}

async function fetchTranscript(videoId: string, config?: TranscriptFetcherConfig): Promise<{
  segments: TranscriptSegment[]
  fullText: string
  language: string
  isAutoGenerated: boolean
}> {
  const url = config?.proxyUrl
    ? `${config.proxyUrl}?videoId=${videoId}`
    : `https://youtubetranscript.com/api/transcript/${videoId}`

  const response = await fetch(url)
  if (!response.ok) {
    throw new Error(`Transcript fetch failed: ${response.status}`)
  }

  const data = await response.json()
  const segments: TranscriptSegment[] = data.map((s: any) => ({
    text: s.text,
    duration: s.duration,
    offset: s.offset,
  }))

  return {
    segments,
    fullText: segments.map(s => s.text).join(' '),
    language: data[0]?.language || 'en',
    isAutoGenerated: data[0]?.isAutoGenerated || false,
  }
}

async function fetchTranscriptWithLanguages(
  videoId: string,
  preferredLanguages: string[] = ['en', 'en-US', 'en-GB'],
  config?: TranscriptFetcherConfig,
): Promise<TranscriptSegment[] | null> {
  for (const lang of preferredLanguages) {
    try {
      const result = await fetchTranscript(videoId, { ...config, lang })
      return result.segments
    } catch {
      continue
    }
  }
  return null
}
```

### 2. Automatic Metadata Extraction

| Change | Location | Description |
|--------|----------|-------------|
| Rich metadata from YouTube oEmbed/Page API | `content-script/videoHelper.ts` | Extract additional metadata beyond title: channel name, publish date, view count, duration, description, category, tags. Use YouTube oEmbed (`https://www.youtube.com/oembed?url=...`) for structured data, or parse the page's `<script type="application/ld+json">`. |
| Topic auto-detection from metadata | `content-script/videoHelper.ts` | Analyze video title, description, and tags against IELTS topic keywords (education, environment, technology, health, etc.). Auto-populate the topic field. |
| Thumbnail URL extraction | `content-script/videoHelper.ts` | Extract video thumbnail URL (`https://img.youtube.com/vi/{videoId}/maxresdefault.jpg`) and store it with the video entry for visual reference in the web app. |
| Duration parsing | `content-script/videoHelper.ts` | Parse ISO 8601 duration from YouTube metadata into human-readable format (e.g., "14:32") and store as `durationMinutes` for filtering/sorting. |

**Implementation Pattern** — expanded `detectVideoPage()` in `videoHelper.ts`:

```typescript
interface RichVideoMeta {
  channelName: string
  channelUrl: string
  publishDate: string
  duration: string // ISO 8601
  durationMinutes: number
  viewCount: number
  description: string
  category: string
  tags: string[]
  thumbnailUrl: string
}

function extractRichMetadata(): RichVideoMeta | null {
  // Try JSON-LD first
  const ldScript = document.querySelector('script[type="application/ld+json"]')
  if (ldScript) {
    try {
      const data = JSON.parse(ldScript.textContent || '')
      const video = Array.isArray(data) ? data.find(d => d['@type'] === 'VideoObject') : data
      if (video?.['@type'] === 'VideoObject') {
        return {
          channelName: video.author?.name || '',
          channelUrl: video.author?.url || '',
          publishDate: video.datePublished || '',
          duration: video.duration || '',
          durationMinutes: parseISODuration(video.duration || ''),
          viewCount: parseInt(video.interactionStatistic?.[0]?.userInteractionCount || '0'),
          description: video.description || '',
          category: video.genre || '',
          tags: video.keywords?.split(',').map((k: string) => k.trim()) || [],
          thumbnailUrl: video.thumbnailUrl?.[0] || `https://img.youtube.com/vi/${getVideoId()}/maxresdefault.jpg`,
        }
      }
    } catch {}
  }
  return null
}

function inferTopicFromMeta(meta: RichVideoMeta): string {
  const text = [
    meta.title,
    meta.description,
    ...(meta.tags || []),
    meta.category,
  ].join(' ').toLowerCase()

  const topicKeywords: Record<string, string[]> = {
    'Education': ['education', 'school', 'university', 'learning', 'student', 'teacher', 'academic', 'study', 'course', 'lecture', 'ielts'],
    'Environment': ['environment', 'climate', 'pollution', 'sustainable', 'ecosystem', 'conservation', 'nature', 'energy'],
    'Technology': ['technology', 'digital', 'internet', 'software', 'ai', 'data', 'computer', 'science', 'innovation'],
    'Health': ['health', 'medical', 'disease', 'treatment', 'hospital', 'patient', 'wellness', 'fitness', 'nutrition'],
    'Economy': ['economy', 'business', 'market', 'finance', 'trade', 'employment', 'income', 'money'],
    'Culture': ['culture', 'society', 'history', 'art', 'music', 'film', 'travel', 'tradition'],
  }

  for (const [topic, keywords] of Object.entries(topicKeywords)) {
    if (keywords.some(k => text.includes(k))) return topic
  }

  return 'General'
}
```

### 3. One-Click Save with Background AI Processing

| Change | Location | Description |
|--------|----------|-------------|
| Combined "Save & Analyze" button | `popup/components/VideoHelper.tsx` | Replace individual AI tab buttons with a single "Save & Analyze" action. Save the video entry immediately with available metadata, then enqueue AI analysis (vocabulary, summary, questions, shadowing) as background jobs. |
| Background analysis queue | `background/videoAnalysisQueue.ts` (new) | Persistent queue for video AI analysis jobs (4 types per video). Processed sequentially per video, in parallel across videos (max 2 videos concurrently). Each analysis type is an independent job. |
| Progressive enhancement in UI | `popup/components/VideoHelper.tsx` | After save, show a progress view: "Saving video... ✓ Analyzing vocabulary... ⏳ Generating summary... ⏳" with real-time updates via `chrome.runtime.sendMessage` from background worker. User can close popup — results appear later in web app. |
| Web app notification | `apps/web/src/services/ProactiveMessageEngine.ts` | When video analysis completes in background, the web app's proactive assistant notifies: "Your video analysis for '{title}' is ready! View vocabulary, summary, and questions." |

**Implementation Pattern** — new file `background/videoAnalysisQueue.ts`:

```typescript
interface VideoAnalysisJob {
  id: string
  videoEntryId: string
  videoTitle: string
  transcript: string
  jobType: 'vocabulary' | 'summary' | 'questions' | 'shadowing'
  status: 'pending' | 'processing' | 'completed' | 'failed'
  error?: string
  createdAt: string
  updatedAt: string
}

class VideoAnalysisQueue {
  private maxConcurrentVideos = 2
  private activeVideoIds = new Set<string>()

  async enqueueAnalysis(entry: VideoEntry, transcript: string): Promise<void> {
    const jobs: VideoAnalysisJob[] = ['vocabulary', 'summary', 'questions', 'shadowing'].map(type => ({
      id: crypto.randomUUID(),
      videoEntryId: entry.id,
      videoTitle: entry.videoTitle,
      transcript,
      jobType: type,
      status: 'pending',
      createdAt: new Date().toISOString(),
      updatedAt: new Date().toISOString(),
    }))

    await this.saveJobs(jobs)
    this.processNext(entry.id)
  }

  private async processNext(videoEntryId: string): Promise<void> {
    if (this.activeVideoIds.size >= this.maxConcurrentVideos) return
    this.activeVideoIds.add(videoEntryId)

    const jobs = await this.getPendingJobs(videoEntryId)
    for (const job of jobs) {
      await this.processJob(job)
      // Notify popup/web app of progress
      chrome.runtime.sendMessage({
        type: 'VIDEO_ANALYSIS_PROGRESS',
        payload: { videoEntryId, jobType: job.jobType, status: 'completed' },
      })
    }

    this.activeVideoIds.delete(videoEntryId)

    // Notify overall completion
    chrome.runtime.sendMessage({
      type: 'VIDEO_ANALYSIS_COMPLETE',
      payload: { videoEntryId },
    })
  }
}
```

## User-Facing Behavior Changes

| Before | After |
|--------|-------|
| Navigate to YouTube → badge appears → click badge → popup opens with empty fields | Navigate to YouTube → transcript auto-fetches in background → badge shows "Analyze" → popup opens with transcript, metadata, and topic pre-filled |
| Must manually copy-paste transcript from YouTube description/CC | Transcript fetched automatically from YouTube captions (auto-generated or manual) |
| Click each AI tab individually (Vocabulary, Summary, Questions, Shadowing) and wait for each to finish | One-click "Save & Analyze" — video saved immediately, all 4 AI analyses run in background |
| Popup must stay open while AI processes (blocks user) | User closes popup immediately after save; analysis continues in background via service worker |
| Topic must be typed manually | Topic auto-detected from video title, description, and tags using keyword matching |
| No channel or duration metadata stored | Rich metadata saved: channel name, publish date, view count, duration, description, tags, thumbnail |
| No way to see analysis status later | Web app shows analysis progress for each saved video; notifies user when complete |
| No auto-save for frequent YouTube users | Opt-in auto-save: visiting a YouTube video auto-saves it with transcript and starts analysis |
| All saved videos look the same in listing | Video library with thumbnails, channel names, duration, and analysis status indicators |

## Performance & Data Considerations

- **Transcript fetch**: Uses free `youtubetranscript.com` API (no API key needed). Falls back to manual paste if unavailable. Cached in session storage per video ID — avoids re-fetch on popup reopen.
- **Analysis queue concurrency**: Max 2 videos processed simultaneously. Jobs within a video run sequentially. Retry with exponential backoff (2s, 10s, 60s). Max 2 retries per job.
- **Metadata extraction**: JSON-LD parsing is O(1) — single DOM query. oEmbed fetch is cached for 1 hour.
- **IndexedDB migration**: DB version bump from 4 to 5. Old entries get default values for new fields (`channelName: ''`, `analysisStatus: {}`, etc.). Backward compatible.
- **Transcript storage**: Full transcript text stored in IndexedDB. For very long videos (>2hrs), transcript may exceed 100KB — stored as a single field, loaded on demand.
- **Auto-save impact**: When enabled, each YouTube navigation triggers transcript fetch (~1 HTTP request) + IndexedDB write. Negligible for normal browsing.

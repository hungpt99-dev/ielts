High-Level Architecture: 2 Approaches

Both keep the overview panel as an iframe communicating via postMessage

Approach A: All-in-One Analysis Message

Single new message type ANALYSIS_DATA returns all overview sections in one response. Content script computes everything (AI + deterministic), sends once.

Pros

  • Simple message flow
  • Panel renders once
  • Easy to cache
  • Minimal iframe complexity

Cons

  • Long wait for everything to load
  • Hard to show partial content
  • AI errors block everything
  • Harder to add/remove sections

Approach B: Section-by-Section Messages

Recommended. Panel requests individual section data. Content script streams results as they're ready. Each section handles its own loading/error state.

Panel → Request: { type: 'REQUEST_OVERVIEW', payload: { sections: ['suitability','vocab','topics','questions'] } }
       ← Section: { type: 'OVERVIEW_SUITABILITY', payload: { ... } }
       ← Section: { type: 'OVERVIEW_VOCAB', payload: { ... } }
       ← Section: { type: 'OVERVIEW_TOPICS', payload: { ... } }
       ← Section: { type: 'OVERVIEW_TECHNICAL', payload: { ... } }  // error in one section

Pros

  • Progressive loading — panels appear as ready
  • One failed section doesn't block others
  • Easy to cache per section
  • Lazy loading — only request visible sections

Cons

  • More message types
  • More complex panel state management
  • Section ordering logic in panel

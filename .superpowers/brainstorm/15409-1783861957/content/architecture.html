Architecture Approaches

With trade-offs for each

A

⭐ Command Pattern (Recommended)

Every edit is a command object: UpdateTaskAction, AddWeekAction, RemovePhaseAction. Edits apply immediately to state, saved to localStorage. Each command is a pure function: (roadmap, params) ⇒ roadmap. Zero dependencies, fully testable, undoable.

Pros

  • Testable — each command is pure
  • Undo/redo built-in
  • Audit trail of changes
  • Type-safe

Cons

  • Slightly more code upfront
  • Overkill if undo not needed
B

Mutable State + Direct Save

Edit mode directly mutates the roadmap object in state. Each field change calls setRoadmap and auto-saves to localStorage. Simpler but mutable.

Pros

  • Less code, faster to build
  • Works with existing patterns

Cons

  • Harder to test
  • No undo
  • Accidental mutations
C

useReducer + Action Dispatch

React useReducer manages roadmap state. Actions dispatched from edit components. Reducer handles all mutations. Pure reducer = testable.

Pros

  • React-idiomatic
  • Actions are serializable
  • Easy undo with history

Cons

  • Reducer file can grow large
  • All state in one reducer
